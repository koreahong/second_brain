# DBT 프로젝트 디렉토리 구조 파악

## 📝 내용

## 목적

- DBT 디렉토리를 파악해서 효율적인 사용을 하기 위함.

## 서칭내용

### 1. analyses

- 역할: SQL 분석 쿼리를 저장하는 공간.

- DBT의 핵심 모델과 별개로, BI 팀에서 데이터를 탐색하거나 시각화에 사용할 수 있는 SQL 파일을 작성하는 곳입니다.

- 쿼리 결과는 데이터베이스에 저장되지 않고, 주로 참고용으로 사용됩니다.

### 2. logs

- 역할: DBT 실행 로그가 저장되는 디렉토리.

- 실행 중 발생한 메시지, 경고, 에러를 추적하여 디버깅하거나 문제를 분석할 수 있습니다.

### 3. macros

- 역할: DBT에서 사용하는 재사용 가능한 SQL 함수(매크로)를 정의하는 공간.

- Jinja 템플릿 언어로 작성되며, 반복되는 작업을 줄이고 SQL 코드를 효율적으로 관리할 수 있습니다.

- 예를 들어, 날짜 포맷팅 함수, 공통 쿼리 필터 등을 여기에 정의합니다.

### 4. models

- 역할: DBT 프로젝트의 핵심 디렉토리로, 데이터 변환 로직을 정의한 SQL 파일들이 저장됩니다.

- 서브폴더:

- DBT가 실행되면 여기 정의된 SQL 파일에 따라 데이터베이스에 테이블 또는 뷰를 생성합니다.

### 5. seeds

- 역할: CSV 파일 형식의 정적 데이터를 저장하는 공간.

- DBT 실행 시 CSV 데이터를 데이터베이스에 로드하여 테이블로 만듭니다.

- 주로 작은 참조 데이터셋(예: 국가 코드, 환율 등)을 관리할 때 사용됩니다.

### 6. snapshots

- 역할: 테이블의 상태 변화를 추적하고 기록하는 공간.

- 데이터베이스의 특정 시점 상태를 저장(snapshot)하고, 변경 이력을 관리합니다.

- 주로 데이터 변경이 빈번한 테이블의 이력 데이터를 분석할 때 사용됩니다.

### 7. target

- 역할: DBT 실행 결과가 저장되는 디렉토리.

- 실행된 SQL 파일의 컴파일 결과, manifest 파일, 실행 로그 등이 포함됩니다.

- 이 디렉토리는 DBT가 실행될 때 자동으로 생성됩니다.

### 8. tests

- 역할: 데이터 품질 검증을 위한 테스트 SQL 또는 YAML 파일을 저장합니다.

- DBT에서는 데이터 무결성 및 정확성을 확인하기 위해 테스트를 작성하며, 여기서 관리합니다.

- 예: 중복 데이터가 없는지, 특정 값이 NULL이 아닌지 확인하는 테스트.

### 9. .gitignore

- 역할: Git 버전 관리에서 제외할 파일과 디렉토리를 정의합니다.

- 일반적으로 target 디렉토리와 같은 실행 결과물은 여기에서 제외 설정합니다.

### 10. dbt_project.yml

- 역할: DBT 프로젝트의 설정 파일.

- 모델, 스냅샷, 시드 등의 경로, 대상 데이터베이스 연결 설정, 및 기타 프로젝트 관련 구성 요소를 정의합니다.

### 11. README.md

- 역할: 프로젝트 설명 문서.

- 팀원이 프로젝트의 목적, 구조, 실행 방법 등을 이해할 수 있도록 문서화합니다.
