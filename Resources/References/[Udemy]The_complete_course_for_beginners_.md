# Data Governance가 흔히 실패하는 이유

1. 정의가 불명확하다 (Not well-defined)

1. 리소스 부족 (Seen as top activity for overstretched resources)

1. 사일로 조직 (Silos, without centralized decision making)

1. 경영진의 이해 부족 (Executives do not understand importance)

1. 프레임워크 미이행 (Framework not followed after implementation)

1. ROI 불명확 (ROI is not clear)

Data Governance VS Data management

Data Governance (데이터 거버넌스)

- “데이터를 어떻게 다뤄야 하는지”를 정하는 정책·규칙·조직 모델

Data Management (데이터 관리)

- “데이터를 실제로 다루는 일상적 활동/프로세스”

1. 내 데이터가 아니고 우리의 데이터라는 것을 인식시켜야 함

데이터 거버넌스 정책은 4가지의 핵심 원리로 이루어져 있고 그것은 7가지 지표로 측정할 수 있음

E level 놀리지 안계 파악 → s level 전략 → tact → Data 전략 파악 → o level 진행

## 1. Executive Level (최고 경영진 층위)

- Steering Committee (운영위원회)

## 2. Strategic Level (전략적 층위)

- Data Governance Council (데이터 거버넌스 협의회)

## 3. Tactical Level (전술적 층위)

- Data Domain Stewards (데이터 도메인 책임자)

- Data Steward Coordinators (데이터 스튜어드 코디네이터)

## 4. Operational Level (실무 층위)

- Data Stewards (데이터 실무자)

## 5. Support Level (지원 조직)

- Data Governance Office / Administrator

- Data Governance Partners (지원 파트너)

- Data Governance Working Teams (작업팀)

## 6. 의사소통 및 승인 경로

- 화살표로 표시된 부분은 Operational → Tactical → Strategic → Executive로 올라가는 보고·승인 경로.

- 반대로 Executive → Operational로는 지침과 정책이 내려옵니다.

- 이 구조 덕분에 데이터 관리가 현장의 업무부터 최고 의사결정까지 일관되게 연결됩니다.

사용하는 데이터는 각 팀에서 담당하게 해야함.→ 얼럿도 보게하고

Task별 업무 거버넌스 파악, 누가 담당이고 책임이 있고 등등 표단위로 볼 수 있게 하는 양식임

## 🔑 전체 구조

- 단계는 Preparation → Setup → Implementation → Maintenance 네 가지 큰 흐름으로 진행됨.

- 각각을 세부 Step 1~10으로 쪼개어 설명한 구조예요.

## 📌 단계별 설명

### Preparation (준비 단계)

Step 1. Identify the Data Governance Leadership & Scope

- 데이터 거버넌스 추진을 누가 리드할지, 그리고 범위(스코프) 를 명확히 정의.

- 예: CDO, Executive Sponsor, Governance Lead 지정.

Step 2. Develop a Data Governance Charter

- 데이터 거버넌스의 비전, 목표, 역할, 책임을 문서화한 Charter(헌장) 작성.

- 조직 전체가 공감할 수 있는 “이유와 방향성” 제공.

### Setup (설정 단계)

Step 3. Choose a model for the Data Governance Team

- 중앙집중형(Centralized), 분산형(Decentralized), 하이브리드 등 운영 모델 결정.

- 조직의 규모, 데이터 환경, 부서 협업 방식에 맞춰 선택.

Step 4. Select the Steering Committee

- 의사결정을 담당할 Steering Committee(운영위원회) 구성.

- 주로 임원진, 각 부서 대표로 구성.

Step 5. Set Up the Data Governance Office

- 실질적 운영을 담당할 Data Governance Office(DGO) 설치.

- 정책 관리, 교육, 커뮤니케이션, 실행 지원 역할.

### Implementation (실행 단계)

Step 6. Choose the Data Governance Working Group

- 각 도메인 전문가, 데이터 스튜어드, IT 담당자 등으로 이루어진 실무 그룹 조직.

- 실제 데이터 관리/품질 개선 활동 수행.

Step 7. Select the Data Governance Support Team
