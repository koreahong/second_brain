# 고급 기능 가이드

## 📝 내용

1. Subagent (Task Tool)

개념: 복잡한 작업을 전문화된 에이전트에게 위임하는 기능입니다.

메인 Claude ──위임──▶ Subagent (특정 작업 수행) ──결과 반환──▶ 메인 Claude

사용 가능한 Subagent 타입:

장점:

- 병렬 실행 가능 (여러 subagent 동시 실행)

- 컨텍스트 분리로 효율적

- 전문화된 검증 가능

1. Hooks

개념: 특정 이벤트 발생 시 자동으로 실행되는 쉘 명령어입니다.

이벤트 발생 ──트리거──▶ Hook 실행 ──결과──▶ Claude에 피드백

Hook 이벤트 타입:

활용 예시:

- 코드 변경 전 자동 린팅

- 파일 저장 후 자동 포맷팅

- 커밋 전 테스트 실행

- 특정 파일 변경 시 알림

1. Slash Commands

개념: / 로 시작하는 사용자 정의 명령어입니다.

/command-name ──확장──▶ 전체 프롬프트로 변환 ──실행──▶ 결과

현재 프로젝트의 Slash Commands:

- /git_push_mr - Git push + MR 생성

- /docker-build-push - Docker 이미지 빌드 및 푸시

- /summarize-troubleshooting - 트러블슈팅 대화 정리

위치: .claude/commands/ 디렉토리에 마크다운 파일로 정의

1. Custom Agents (Validators)

개념: 특정 도메인에 특화된 검증 에이전트입니다.

코드/설정 ──검증 요청──▶ Validator Agent ──검증 결과──▶ 문제점/권장사항

1. Plan Mode

개념: 복잡한 작업 전 계획을 수립하고 승인받는 모드입니다.

복잡한 요청 ──▶ Plan Mode 진입 ──▶ 코드베이스 탐색 ──▶ 계획 수립 ──▶ 사용자 승인 ──▶ 구현

사용 시점:

- 여러 접근 방식이 있을 때

- 아키텍처 결정이 필요할 때

- 대규모 변경 작업

- 요구사항이 불명확할 때

1. TodoWrite (작업 관리)

개념: 작업을 계획하고 진행 상황을 추적하는 도구입니다.

작업 분해 ──▶ Todo 리스트 생성 ──▶ 순차적 실행 (in_progress) ──▶ 완료 표시

상태: pending → in_progress → completed
