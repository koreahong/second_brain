# 2025년 10월 27일

## 🚩 문제 상황

• 인원이 적고 변동이 잦은 팀 특성상, 코드 스타일·작업 방식이 일관되지 않아 생산성이 흔들림.
• 각 팀 간 데이터 공유에 대한 인식 차이로 협업이 어려움.
    ◦ 특히 일부 팀은 데이터 공유를 ‘IP 노출 리스크’로 인식하여 비협조적.
    ◦ 거버넌스의 필요성과 장점을 체감하지 못하고 있음.

## 🔧 대처 방법

• Claude Code 도입: 코드 컨벤션, 리뷰, 문서화를 자동화해 일관성 강화.
• Jira, Confluence 연동: 작업 추적과 문서화를 자동화해 협업 효율화.
• FactSet 데이터 계약 협의: 경쟁사 대비 낮은 단가로 계약 유도.
• 데이터 공유 설득:
    ◦ 데이터팀이 중앙에서 관리할 때의 이점(품질·보안·중복 방지)을 강조.
    ◦ 공유 지표 및 메타데이터를 통해 ‘누가 어떤 데이터를 어떻게 쓰는지’ 명확히 관리 가능함을 설명.

## ✨ 성과 / 개선점

성과:
• Claude Code 도입으로 신규 인원도 빠르게 개발 환경에 적응 가능.
• Jira–Confluence 통합으로 작업 투명성과 히스토리 관리 향상.
• FactSet 계약비용 절감(경쟁사 대비 유리한 단가 확보).
• 일부 조직이 데이터 공유의 필요성과 가치에 공감하기 시작.
개선점:
• 여전히 ‘데이터 IP 노출’에 대한 불안감이 남아 있음.
• 공유된 데이터의 사용 추적 체계(DataHub, lineage tagging 등) 추가 필요.

## 🤝 배운 점

• 기술보다 중요한 것은 **‘데이터를 신뢰할 수 있는 문화’**를 만드는 것임.
• 거버넌스는 강제보다 설득과 신뢰를 기반으로 해야 지속 가능함.

## 💡 한 줄 요약

“기술적 일관성과 데이터 신뢰 문화를 동시에 구축한 전환점이었다.”
