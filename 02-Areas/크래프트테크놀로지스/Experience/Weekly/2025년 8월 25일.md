# 2025년 8월 25일

## 🚩 문제 상황

• 팀별 데이터 사용·적재 현황이 문서화되지 않아, 어떤 데이터를 누가 어떻게 관리·활용하는지 파악하기 어려움.
• 서비스로 나가는 데이터조차 출처나 가공 로직이 불분명한 경우가 있어 거버넌스 리스크가 존재.

## 🔧 대처 방법

• 문서화 체계 배포
    ◦ 팀별 소통 채널을 통해 컨플루언스 양식을 배포:
        ▪ 사용현황, 적재상황, 구매내역
        ▪ 테이블 정의서, 컬럼 정의, 가공 로직
    ◦ 전체 공지를 통해 참여 독려.

## ✨ 성과 / 개선점

• 성과: 각 팀이 자체적으로 데이터 사용현황 페이지를 작성하기 시작.
• 개선: 작성 수준과 관리 지속성은 팀별 편차가 커서, 전사적 표준화가 필요.

## 🤝 배운 점

• 각 팀에서 데이터 관리에 대한 내재적 수요가 크다는 것을 확인.
• 이 수요를 적극적으로 자극해, 각 팀이 스스로 관리해야 한다는 분위기를 만드는 것이 중요.

## 💡 한 줄 요약

• 팀별 문서화 양식을 배포해 데이터 관리 수요를 끌어내고, 자체 관리 분위기 조성의 기반을 만들었다.

## 역량

#문서화 #의사소통 #구조화 #운영 체계화


## 추가 내용

### what

팀별 데이터관련 문서 작성 양식 배포

### how

- 팀별 소통 채널 + 컨플루언스 양식 배포(사용현황, 적재상황, 구매 등), 테이블 정의서, 컬럼, 가공 로직 
- 전체 공지
### result

- 팀별 사용현황 페이지 작성
### 느낀점

- 각 팀에서 데이터 관리에 대한 수요가 많음을 알 수 있었음.
- 해당 수요를 자극해서 자체적으로 관리해야하는 분위기를 만들어야 할 것임
