## 젠킨스 서치

### 젠킨스란

- 레포지토리에 대한 CI(지속적인 통합)과 CD(지속적 배포)의 프로세스(빌드, 테스트, 배포) 를 자동화하는 도구
- github action과 같은 역할
### 사용하는 이유

- 편리한 설정
  - 웹기반, 플러그인 사용
  - 빌드/배포에 대한 통지를 받음
- 안정적인 빌드/배포 환경
  - 유닛, 통합 테스트 자동화
### 기능

- git에서 commit을 감지하면 자동적으로 빌드가 작동
### 연관용어

- jar: java Archive의 약자로 자바에서 사용되는 압축 파일의 한 형태 #jar
- war #war
  - Web Application Archive의 약자로 웹 애플리케이션을 압축하고 배포하는데 사용되는 파일 형태.
  - WAR는 JSP, Servlet, Java Class, XML, 라이브러리, 정적 웹페이지(html ...) 및 웹 애플리케이션을 구성할 때 필요한 자원을 압축한 jar 파일
  - WEB-INF 및 META-INF 디렉토리로 사전 정의된 구조를 사용하며 WAR 파일을 실행하려면 Tomcat과 같은 웹서버 또는 웹 컨테이너가 필요
- WEB-INF #WEB-INF
  - 외부에서 접근할 수 없는 웹정보가 있는 폴더
- META-INF #META-INF
  - Java에서 설정관련 파일을 저장하는 폴더
- Apache #Apache
  - 아파치 소프트웨어 재단에서 관리하는 HTTP 웹 서버
  - 클라이언트가 GET, POST 등의 메소드를 이용해 전송한 요청을 받아 HTML이나 오브젝트를 HTTP 프로토콜을 이용해 전송하는 프로그램
- 웹서버 #웹서버
  - 네트워크 통신을 구현하기 위해 미리 만들어진 프로그램을 사용하는데 웹 서버와 웹 브라우저
- Tomcat #Tomcat
  - 아파치 재단에서 만든 오픈 소스 WAS(Web Application Server)
  - Tomcat은 Java Servlet과 JSP가 실행할 수 있는 환경을 제공하여 동적인 페이지를 생성
  - Tomcat의 디렉토리 구조
    - Bin: 톰캣 서버의 동작을 제어할 수 있는 스크립트 및 실행 파일
    - Conf: 톰캣의 기본적인 설정 파일
    - Lib: 아파치와 같은 다른 웹 서버와 톰캣을 연결해주는 바이너리 모듈들
    - Webapps: 톰캣이 제공하는 웹 애플리케이션의 기본 위치
    - Logs: 서버의 로그 파일이 저장
    - Work: jsp 컨테이너와 다른 파일들이 생성하는 임시 디렉토리
    - Temp: 임시 저장 폴더
- WAS #WAS
  - 웹 서버와 웹 컨테이너의 결합으로 이루어진 소프트웨어
  - 다양한 클라이언트의 요구에 유연하게 대처하기 위해 DB와 연결해 데이터를 주고받거나 데이터 조작을 하여 동적인 페이지를 생성해 응답하기 위해 WAS를 사용
- Servlet #Servlet
  - 서블릿이란 자바를 사용하여 웹을 만들기 위해 필요한 기술
  - 클라이언트가 어떠한 요청을 하면 그에 대한 결과를 다시 전송
  - 웹서버가 동적인 페이지를 제공할 수 있도록 도와주는 어플리케이션이 서블릿이며, 동적인 페이지를 생성하는 어플리케이션이 CGI입니다
- JSP #JSP
  - Java 코드가 들어가 있는 HTML 코드
- 컴파일 #컴파일
  - 컴파일이란 개발자가 작성한 소스코드를 바이너리 코드로 변환하는 과정
  - 컴퓨터가 이해할 수 있는 기계어로 변환하는 작업
- 링크 #링크
  - 컴파일과정을 통해 만들어진 목적 파일을 실행파일로 만드는 것
- 빌드 #빌드
  - 소스 코드 파일을 실행 가능한 형태로 변환하는 과정
- 배포 #배포
  - 빌드가 완성된 실행 가능한 파일 사용자가 접근할 수 있는 환경에 배치시키는 일
