## 📊 PCAP vs Iceberg(같은 거버넌스 레이어) 비교

### PCAP 기반 처리

- ✅ 장점:
  - 네트워크 패킷 그대로라서 정밀한 호가/체결 데이터를 손실 없이 재현 가능.
  - 초저지연/틱 단위 분석에 최적.
- ❌ 단점:
  - 파일 단위 관리라서 스키마·메타데이터 부재 → 재사용성 낮음.
  - 스토리지·인덱싱 비효율 → 장기 보관·검색 어려움.
  - 팀별로 다른 방식으로 파싱 → 일관성 부족.
---

### Iceberg/Hudi/Delta Lake 같은 거버넌스 레이어

- ✅ 장점:
  - 스키마 관리 + 버저닝 + 파티셔닝 → 특정 기간/종목만 빠르게 조회 가능.
  - SQL/Trino/Spark/dbt 등 친숙한 도구로 액세스 가능 → 리서처들이 Python, SQL로 더 쉽게 사용.
  - 데이터 품질/라인리지 관리 → 재현성 있는 백테스트 가능.
  - S3 같은 오브젝트 스토리지에 쌓으면 장기 보관·비용 최적화.
- ❌ 단점:
  - 초저지연 틱단위 실험에는 부적합 (쿼리 레이어 오버헤드 존재).
  - 세밀한 마이크로초 단위 시뮬레이션 → 여전히 PCAP/raw 접근 필요.
  - 초기 구축 비용·운영 복잡성 존재.
