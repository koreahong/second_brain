### 빌드

- 컴파일 -> 링크를 과정을 통틀어 일컫는 말
- 소스코드를 실행가능한 형태로 변환하는 말
### 컴파일

- 소스코드를 컴퓨터가 이해할 수 있는 바이너리 파일로 변환하는 형태이다.
- java는 jvm, python -> pyc가 있다.
### 링크

- 목적파일 -> 실행파일로 만드는 것
### 내용

- python은 따로 jar파일이 만들어지지 않는 이유는 python은 인터프리터로 소스코드가 직접 실행되기 때문임
### 컴파일러 VS 인터프리터

### 컴파일러:

컴파일러는 전체 소스 코드를 컴파일하고 실행 파일로 변환하는 프로그램입니다. 이 변환은 소스 코드의 전체 구조를 이해하고 목적 코드로 번역합니다. 번역된 목적 코드는 특정 플랫폼에서 직접 실행할 수 있는 이진 파일이거나 중간 언어 (바이트 코드)일 수 있습니다.

1. 장점:
  - 실행 속도가 빠릅니다. 번역된 코드는 이미 플랫폼에 최적화되어 있기 때문입니다.
  - 컴파일 후에는 원본 소스 코드가 필요 없습니다.
1. 단점:
  - 컴파일 시간이 오래 걸릴 수 있습니다.
  - 변경된 내용을 반영하기 위해서는 다시 컴파일해야 합니다.
  - 플랫폼 간 이식성이 낮을 수 있습니다.
### 인터프리터:

인터프리터는 소스 코드를 한 줄씩 읽고 실행하는 방식입니다. 소스 코드를 직접 실행하면서 동시에 번역하는 방식이며, 특별한 실행 파일이나 목적 코드를 생성하지 않습니다.

1. 장점:
  - 코드 수정 후 즉시 실행할 수 있습니다.
  - 플랫폼 간 이식성이 높습니다.
  - 디버깅이 상대적으로 쉽습니다.
1. 단점:
  - 실행 속도가 컴파일에 비해 느립니다.
  - 전체 소스 코드가 존재해야 실행이 가능합니다.
  - 일반적으로 컴파일러에 비해 성능이 낮을 수 있습니다.
### pyc 파일을 생성하는 이유

1. 성능 향상: 소스 코드를 직접 실행하는 것보다 바이트 코드를 사용하는 것이 성능상 이점이 있습니다. 컴파일된 바이트 코드는 인터프리터에 비해 빠르게 실행될 수 있습니다.
1. 캐싱: .pyc 파일은 소스 코드를 컴파일한 결과물을 저장하는 캐시 역할을 합니다. 이는 프로그램을 다시 실행할 때 동일한 모듈이 다시 컴파일되지 않고, .pyc 파일에서 바로 로드되어 속도를 향상시킵니다.
1. 코드 숨김: 소스 코드를 바이트 코드로 변환하고 컴파일된 .pyc 파일을 배포함으로써, 소스 코드를 감추어 코드의 일부를 보호할 수 있습니다. 하지만 이것은 보안 기능이 아니며, Python의 특성상 코드가 어느 정도 읽기 쉬운 형태로 남아있습니다.
1. 결론: 이러한 이유로, Python은 소스 코드를 직접 실행하는 대신에 컴파일된 바이트 코드인 .pyc 파일을 활용하여 프로그램을 실행하는 방식을 채택합니다.
