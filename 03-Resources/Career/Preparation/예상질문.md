퇴사사유

- 데이터 엔지니어로서 성장
- 에이브랩스
데이터 안정적 수집

- API
  - 요청해야 하는 API 일일 DB저장 후 기록
  - API 초당 2회 제한
  - 파라미터 경우의 수
  - 토큰 갱신 로직
  - sqlalchemy로 데이터 모델에 맞추고 스키마 변동시 얼럿
- 크롤링
  - DOM 구조 변경시 얼럿
  - 테스트 케이스
  - id/class 값을 config 파일로 관리
품질 관리 자동화

- GX 사용
  - 매일 검증 리포트를 S3로 호스팅해서 확인
  - 검증에 안맞는 이슈가 있을시 얼럿
데이터 마트 설계

- 비지니스 요건에 맞게 최대한 설계
- ERD 작성, data.io 사용
데이터 파이프라인 견고하게

- 데이터나 마트관련 요건 수정 - openmetadata
  - 데이터 흐름 관리
- airflow 서버나 dag에 대한 모니터링 - grafana
- 얼럿 발생시에 어떤 부분을 봐야하는지 상세하게 메세지 남김
금융 도메인에 대한 이해

- 프로젝트를 진행하면서 여러 도메인에 대한 이해를 해야했음
db 최적화

- 인덱스, range 파티셔닝, 클러스터링
- 최근 patial index
전처리

- null 값 대체
질문

- 전처리나 마트 설계 단계까지 데이터 파이프라인에서 발생하고 있는 병목 지점이나 기술적 어려움이 있다면 어떤 부분인지 여쭤보고 싶습니다.”
- 다루고 있는 데이터 규모 
- 입사 초기 데이터 엔지니어로서 우선적으로 해결하기 바라는 부분
