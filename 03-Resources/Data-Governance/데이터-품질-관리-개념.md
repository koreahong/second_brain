# 데이터 품질 관리 개념

## 📋 정의

데이터 품질(Data Quality)은 데이터가 의도된 목적에 얼마나 적합한지를 나타내는 지표입니다. 데이터 품질 관리는 이러한 품질을 측정, 모니터링, 개선하는 체계적인 프로세스입니다.

## 🎯 품질 차원 (Quality Dimensions)

### 1. 정확성 (Accuracy)
**정의**: 데이터가 실제 값이나 참 값을 올바르게 반영하는 정도

**측정**:
- 오류율 = (오류 건수 / 전체 건수) × 100
- 정확도 = 100 - 오류율

**예시**:
- ✅ 고객 이메일 주소가 실제 이메일과 일치
- ❌ 전화번호에 잘못된 자릿수

### 2. 완전성 (Completeness)
**정의**: 필수 데이터가 모두 존재하는 정도

**측정**:
- 완전성 = (채워진 필드 / 전체 필수 필드) × 100

**예시**:
- ✅ 모든 주문에 고객 ID 존재
- ❌ 일부 제품에 가격 정보 누락

### 3. 일관성 (Consistency)
**정의**: 같은 데이터가 여러 곳에서 동일한 형태로 표현되는 정도

**측정**:
- 중복 데이터 비율
- 형식 불일치 비율

**예시**:
- ✅ 날짜가 모두 'YYYY-MM-DD' 형식
- ❌ 고객명이 시스템마다 다르게 표기

### 4. 적시성 (Timeliness)
**정의**: 데이터가 필요한 시점에 이용 가능한 정도

**측정**:
- 데이터 신선도 = 현재 시간 - 최종 업데이트 시간
- SLA 준수율

**예시**:
- ✅ 실시간 주가 데이터가 5초 이내 업데이트
- ❌ 일일 리포트가 2일 지연

### 5. 유효성 (Validity)
**정의**: 데이터가 정의된 형식, 타입, 범위 규칙을 준수하는 정도

**측정**:
- 규칙 위반 비율

**예시**:
- ✅ 나이 필드가 0-150 범위 내
- ❌ 이메일 필드에 '@' 없음

### 6. 고유성 (Uniqueness)
**정의**: 중복 없이 각 데이터가 고유한 정도

**측정**:
- 중복률 = (중복 건수 / 전체 건수) × 100

**예시**:
- ✅ 주문 ID가 모두 고유
- ❌ 동일 고객이 여러 번 등록

## 🏗️ 품질 관리 프로세스

### 1. 측정 (Measure)
**활동**:
- 품질 기준 정의
- 품질 메트릭 수집
- 베이스라인 설정

**도구**:
- Data Profiling
- Statistical Analysis
- Quality Dashboards

### 2. 모니터링 (Monitor)
**활동**:
- 실시간 품질 체크
- 임계값 기반 알림
- 트렌드 분석

**방법**:
- 배치 검증
- 실시간 검증
- 샘플링 검증

### 3. 분석 (Analyze)
**활동**:
- 근본 원인 파악
- 영향도 분석
- 패턴 식별

**기법**:
- Root Cause Analysis
- Pareto Analysis
- Impact Assessment

### 4. 개선 (Improve)
**활동**:
- 품질 이슈 해결
- 프로세스 개선
- 예방 조치

**방법**:
- 데이터 정제
- 소스 시스템 수정
- 검증 규칙 추가

## 📊 품질 관리 기법

### 1. Data Profiling
**목적**: 데이터 특성 및 품질 현황 파악

**분석 항목**:
- 데이터 타입 분포
- Null 값 비율
- 고유값 개수
- 통계적 분포 (평균, 표준편차, 최소/최대)
- 패턴 및 이상치

### 2. Data Validation
**유형**:

**Schema Validation**:
- 컬럼 존재 여부
- 데이터 타입 일치
- Not Null 제약

**Business Rule Validation**:
- 값 범위 검증
- 참조 무결성
- 논리적 일관성

**Cross-System Validation**:
- 시스템 간 데이터 일치
- 합산 검증

### 3. Data Cleansing
**기법**:

**표준화 (Standardization)**:
- 형식 통일 (예: 전화번호)
- 대소문자 정규화
- 공백 제거

**파싱 (Parsing)**:
- 주소 분리 (도로명, 상세 주소)
- 이름 분리 (성, 이름)

**보정 (Correction)**:
- 오타 수정
- 주소 검증 및 보정
- 표준 코드 매핑

**중복 제거 (Deduplication)**:
- Exact Match
- Fuzzy Match
- Phonetic Match

### 4. Data Quality Metrics
**주요 지표**:

```
품질 점수 = Σ(차원별 품질 × 가중치) / Σ가중치

예시:
정확성 (30%) + 완전성 (25%) + 일관성 (20%) +
적시성 (15%) + 유효성 (10%) = 100%
```

## 🚨 일반적인 문제와 해결

### 문제 1: 품질 기준 모호
**증상**: 주관적 판단, 일관성 부족

**해결**:
- 명확한 품질 규칙 정의
- 수치화 가능한 메트릭
- 문서화 및 공유

### 문제 2: 품질 체크 부담
**증상**: 성능 저하, 개발 지연

**해결**:
- 샘플링 활용
- 증분 검증
- 비동기 처리

### 문제 3: False Positive 많음
**증상**: 잘못된 알림 빈발

**해결**:
- 임계값 조정
- 예외 규칙 정의
- 점진적 엄격화

### 문제 4: 품질 이슈 반복
**증상**: 같은 문제 재발

**해결**:
- 근본 원인 해결
- 소스 시스템 개선
- 예방 규칙 추가
