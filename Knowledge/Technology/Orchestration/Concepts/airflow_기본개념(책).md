# airflow 기본개념(책)

## 📝 내용

에어플로우는 일련의 테스크를 비순환 그래프라는 개념을 도입하여 파이프라인을 구현한 것. 일련의 테스크를 모은 것을 dag라는 단위로 관리한다.

에어플로우 구조

배치로 처리한다는 것은 실시간 데이터를 보낸다는 것이 아닌, 일정기간의 데이터를 쌓아두고 처리한다는 것

template 사용

execution_date는 dag가 실제 실행되는 순간이 아니라 예약 간격을 표시하는 것

팬인 

팬아웃: 여러 개의 입력 태스크 연결 수 제한

- 팬아웃 종속성: 한 태스크를 여러 다운스트림 태스크에 연결하는 것

Xcom은 테스크간 데이터를 전달해주기 위한 매개체 역할
pythonopertor에서 결과값을 리턴해서 다른 테스크에서 해당 결과값을 받아서 사용할 수 있도록 설계할 수 있음

오케스트레이션 시스템에서 다른 시스템과연결 하기 위해서 훅으로 통신을 하는데
